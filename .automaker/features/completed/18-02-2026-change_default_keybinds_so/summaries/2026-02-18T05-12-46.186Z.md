## Summary: Add Keyboard Shortcuts for Board Panel Toggle (Ctrl+Q/W/E/R)

### Changes Implemented

- Added 4 new keyboard shortcuts to collapse/expand panels in the kanban board view:
  - **Ctrl+Q** - Toggle File Explorer panel
  - **Ctrl+W** - Toggle Kanban Board panel
  - **Ctrl+E** - Toggle Agent Chat panel
  - **Ctrl+R** - Toggle Deploy panel
- Added shortcut definitions to the `KeyboardShortcuts` interface and `DEFAULT_KEYBOARD_SHORTCUTS` constant
- Extended the `useBoardKeyboardShortcuts` hook to accept and register the 4 new panel toggle callbacks
- Wired up the toggle callbacks in `board-view.tsx` using the existing panel state (local `showFileExplorer` state and app store collapse states)
- Added the new shortcuts to the keyboard map UI so they appear in the shortcut reference panel with proper labels and categorization

### Files Modified

- `apps/ui/src/store/app-store.ts` - Added `toggleFileExplorer`, `toggleKanbanPanel`, `toggleAgentChat`, `toggleDeployPanel` to `KeyboardShortcuts` interface and `DEFAULT_KEYBOARD_SHORTCUTS`
- `apps/ui/src/components/views/board-view/hooks/use-board-keyboard-shortcuts.ts` - Added 4 new optional callbacks to the props interface and registered them as keyboard shortcuts
- `apps/ui/src/components/views/board-view.tsx` - Passed panel toggle callbacks to the keyboard shortcuts hook
- `apps/ui/src/components/ui/keyboard-map.tsx` - Added labels and categories for the new shortcuts so they display in the keyboard reference UI

### Notes for Developer

- The shortcuts use `Ctrl+` modifier which prevents conflicts with existing single-key shortcuts (Q for cyclePrevProject, E for cycleNextProject, etc.)
- `Ctrl+W` and `Ctrl+R` will override the browser's default close-tab and refresh behavior when the board view is active, since `event.preventDefault()` is called. This is intentional per the user's request and consistent with the Electron desktop app context.
- Shortcuts are only active in the board view and respect the existing `isInputFocused()` guard (won't fire when typing in inputs or when dialogs are open)
- Shortcuts are customizable through the existing keyboard shortcuts settings UI
