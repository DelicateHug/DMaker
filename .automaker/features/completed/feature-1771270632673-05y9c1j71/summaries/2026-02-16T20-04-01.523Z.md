## Summary: Fix crash when feature.json is missing from feature directory

### Problem

The app was crashing with "Something Went Wrong" error screen when a feature directory existed but its `feature.json` file was missing or unrecoverable. The AtomicWriter logged warnings about failed recovery, but the server-side handling was mostly correct. The main issues were:

1. **Noisy warnings**: Every time features loaded, the AtomicWriter logged `WARN` messages for orphaned feature directories (no `feature.json` and no recovery files), cluttering logs
2. **Missing field defaults**: Features loaded from disk could have `undefined` for `description` or `category`, causing client-side crashes when code called `.toLowerCase()` on these fields
3. **Client-side null safety**: Several UI components accessed feature properties without null guards, causing render crashes caught by the `RouteErrorBoundary`

### Changes Implemented

**Server-side (feature-loader.ts):**

- Added early orphan detection in `getAll()` and `getAllSummaries()`: checks if `feature.json` exists before invoking the full recovery flow. If the directory has no `feature.json` and no temp/backup recovery files, it's silently skipped with a `debug` log instead of noisy `WARN` logs
- Added safe defaults for required string fields after loading: `description` defaults to `''` and `category` defaults to `'Uncategorized'` to prevent downstream crashes
- Applied the same `category` default in `getAllSummaries()` summary extraction

**Client-side (UI):**

- `use-board-column-features.ts`: Fixed search filter to use `(f.title || '').toLowerCase()` and `(f.description || '').toLowerCase()` instead of bare `.toLowerCase()`. Also added `title` to the search fields (was previously only searching `description` and `category`)
- `use-board-features.ts`: Added `description: ''` and `category: 'Uncategorized'` defaults in both Phase 2 feature loading paths (single project and all-projects mode), and in `summaryToFeature()`
- `list-row.tsx`: Fixed `getFeatureSortValue()` to use `(feature.title || feature.description || '').toLowerCase()` to prevent crash when both fields are undefined

### Files Modified

- `apps/server/src/services/feature-loader.ts` - Orphaned directory detection + safe field defaults
- `apps/ui/src/components/views/board-view/hooks/use-board-column-features.ts` - Null-safe search filter
- `apps/ui/src/components/views/board-view/hooks/use-board-features.ts` - Safe defaults in feature loading
- `apps/ui/src/components/views/board-view/components/list-view/list-row.tsx` - Null-safe sort value

### Notes for Developer

- All 65 feature-loader unit tests pass
- The 7 test failures in the full server test suite are pre-existing (cursor provider tests and Windows file locking in worktree integration tests)
- The orphaned directory detection uses `logger.debug()` instead of `logger.warn()` to reduce log noise for a common scenario (e.g., incomplete feature deletion)
- If recovery files (`.tmp.*` or `.bak*`) exist alongside a missing `feature.json`, the full AtomicWriter recovery flow is still attempted
