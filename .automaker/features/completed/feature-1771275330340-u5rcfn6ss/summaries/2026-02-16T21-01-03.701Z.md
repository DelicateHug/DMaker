## Summary: TypeError: Cannot read properties of undefined (reading 'onDeployOutput')

### Root Cause

The `deploy-panel.tsx` component accesses `client.deploy.onDeployOutput()`, `client.deploy.onDeploySuccess()`, and `client.deploy.onDeployError()` on the `HttpApiClient` instance as a WebSocket fallback for SSE streaming. However, the `HttpApiClient` class had no `deploy` property defined, so `client.deploy` was `undefined`, causing the TypeError when the component's `useEffect` hook ran.

Additionally, the `EventType` union in `http-api-client.ts` was missing the `deploy:output`, `deploy:success`, and `deploy:error` event types (even though they were already defined in `@dmaker/types/event.ts`).

### Changes Implemented

1. **Added deploy event types to the local `EventType` union** in `http-api-client.ts` ‚Äî added `'deploy:output'`, `'deploy:success'`, and `'deploy:error'` entries
2. **Added `deploy` property to `HttpApiClient` class** with three WebSocket event subscription methods:
   - `onDeployOutput(callback)` ‚Äî subscribes to `deploy:output` events (payload: `{ data: string }`)
   - `onDeploySuccess(callback)` ‚Äî subscribes to `deploy:success` events (payload: `{ message?: string; duration?: number }`)
   - `onDeployError(callback)` ‚Äî subscribes to `deploy:error` events (payload: `{ message?: string; error?: string; duration?: number }`)

### Files Modified

- `apps/ui/src/lib/http-api-client.ts` ‚Äî Added deploy event types to `EventType` union (line ~549) and added `deploy` property to `HttpApiClient` class (line ~3008)

### Notes for Developer

- The payload types for the deploy event subscription methods were inferred from the actual usage in `deploy-panel.tsx` to ensure type compatibility.
- The deploy panel's primary streaming mechanism is SSE (in `handleRunScript`); the WebSocket subscriptions serve as a fallback when SSE doesn't connect. This is consistent with how the component was designed ‚Äî the `sseReceivedDataRef` flag prevents duplicate processing when SSE is working.
- The server's deploy route (`apps/server/src/routes/deploy/routes/run.ts`) currently only streams via SSE, not WebSocket. If WebSocket fallback is needed end-to-end, the server would also need to broadcast deploy events over WebSocket, but that's a separate concern ‚Äî this fix prevents the client-side crash.
