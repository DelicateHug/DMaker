## Summary: Startup Performance & Unused Dependencies Optimization

### Changes Implemented

1. **Removed unused `SplashScreen` component** (`apps/ui/src/components/splash-screen.tsx`)
   - The component was no longer imported or rendered anywhere in the application
   - Tests reference `dmaker-splash-shown` sessionStorage key but handle its absence gracefully

2. **Lazy-loaded heavy conditionally-rendered components in `__root.tsx`**
   - `VoiceWidget` — only shown when voice mode is enabled; now loaded on demand via `React.lazy()`
   - `SandboxRiskDialog` — only shown when sandbox confirmation needed; now lazy-loaded with guard (`{showSandboxDialog && <Suspense>...}`)
   - `SandboxRejectionScreen` — only shown when user denies sandbox risk; now lazy-loaded
   - All wrapped in `<Suspense fallback={null}>` for seamless loading

3. **Removed 16 unused `@fontsource` packages from UI dependencies**
   - Removed: `@fontsource/cascadia-code`, `@fontsource/fira-code`, `@fontsource/ibm-plex-mono`, `@fontsource/inconsolata`, `@fontsource/inter`, `@fontsource/iosevka`, `@fontsource/jetbrains-mono`, `@fontsource/lato`, `@fontsource/montserrat`, `@fontsource/open-sans`, `@fontsource/poppins`, `@fontsource/raleway`, `@fontsource/roboto`, `@fontsource/source-code-pro`, `@fontsource/source-sans-3`, `@fontsource/work-sans`
   - None of these packages were imported anywhere in the source code
   - The app uses only bundled Zed fonts (from local CSS files) and system fonts
   - Reduces `npm install` time and `node_modules` size

### Pre-existing Optimizations Verified (from previous work)

- All route components use `lazyRouteComponent()` for code splitting
- Theme CSS is lazily loaded via `lib/theme-loader.ts` (only dark/light loaded synchronously)
- Font preloading is deferred by 2 seconds via `font-imports.ts`
- Cursor status check is deferred by 3 seconds
- Provider auth status check is deferred by 2 seconds
- Auth state is cached optimistically to avoid blocking on server verification
