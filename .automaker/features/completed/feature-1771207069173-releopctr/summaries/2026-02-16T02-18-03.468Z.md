## Summary: All scripts should be given an id of project-mmhhddmmyyyy

### Changes Implemented

- Created a centralized `generateProjectId()` utility function that produces project IDs in the format `project-MMHHddmmYYYY` (e.g., `project-021410302026` for Feb 14th at 10:30, 2026)
- Replaced all 11 instances of `project-${Date.now()}` in production code with calls to `generateProjectId()`
- The format breakdown: MM (month), HH (hour), dd (day), mm (minute), YYYY (year)

### Files Modified

- **`apps/ui/src/lib/utils.ts`** — Added the `generateProjectId()` function after the existing `generateUUID()` function
- **`apps/ui/src/store/app-store.ts`** — Added import and replaced 1 usage in `openProject()`
- **`apps/ui/src/components/views/welcome-view.tsx`** — Added import and replaced 3 usages (blank project, template project, existing project creation)
- **`apps/ui/src/components/views/dashboard-view.tsx`** — Added import and replaced 3 usages (blank, template, existing project creation)
- **`apps/ui/src/components/views/interview-view.tsx`** — Added import and replaced 1 usage (interview-based project creation)
- **`apps/ui/src/components/views/settings-view/projects/projects-section.tsx`** — Added import and replaced 3 usages (blank, template, existing project creation)

### Notes for Developer

- Test infrastructure files (`tests/utils/project/setup.ts`, test specs) were intentionally left using `Date.now()` for project IDs since they need guaranteed uniqueness across parallel test runs and are not user-facing
- The new format has minute-level precision. If two projects are created within the same minute, they would receive the same ID. In production usage this is unlikely since project creation is a manual user action. If higher precision is needed, seconds could be appended to the format.
- Existing projects with old `project-{timestamp}` IDs will continue to work — the change only affects newly created projects
