## Summary: Alt+Enter Support and Button Label Update

### Changes Implemented

1. **Alt+Enter Keyboard Shortcut Support**: Added Alt+Enter keyboard shortcut to submit the "Add Feature" dialog from the description textarea
   - Users can now press Alt+Enter while typing in the description field to quickly add a feature to the backlog
   - This provides a faster workflow alongside the existing Cmd/Ctrl+Enter shortcut on the button

2. **Button Label Update**: Changed the "Add Feature" button text to "Add to Backlog"
   - More clearly indicates that features are being added to the backlog column
   - Maintains the "Spawn Task" label when in spawn mode

3. **Test Coverage**: Added comprehensive E2E test for the Alt+Enter functionality
   - Verifies that Alt+Enter correctly submits the form
   - Ensures the feature appears in the backlog column after submission

### Files Modified

- `apps/ui/src/components/ui/description-image-dropzone.tsx:51-52,71,386-404,448` - Added `onAltEnter` prop and keyboard handler
- `apps/ui/src/components/views/board-view/dialogs/add-feature-dialog.tsx:639,939` - Connected Alt+Enter handler and updated button label
- `apps/ui/tests/features/add-feature-to-backlog.spec.ts:91-124` - Added E2E test for Alt+Enter shortcut

### Notes for Developer

- The Alt+Enter shortcut only triggers when:
  - The textarea has focus
  - Alt key is pressed with Enter (no other modifiers like Shift/Ctrl/Cmd)
  - The dialog is open and not disabled
- The existing Cmd/Ctrl+Enter shortcut on the button continues to work as before
- The button now shows "Add to Backlog" which better reflects the action being performed (adding to the backlog column)

- Both keyboard shortcuts (Alt+Enter and Cmd/Ctrl+Enter) will submit the same form action via the `handleAdd` function
